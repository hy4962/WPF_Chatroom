# 单例模式详解

## 一、什么是单例模式？

**单例模式（Singleton Pattern）** 是一种设计模式，确保一个类**只有一个实例**，并提供一个**全局访问点**来获取这个实例。

```
普通模式：每次创建都是新对象
new HomeVM() → 对象1
new HomeVM() → 对象2（不同的对象）

单例模式：永远只有同一个对象
HomeVM.Instance → 对象1
HomeVM.Instance → 对象1（同一个对象）
```

## 二、为什么需要单例模式？

### 适用场景

1. **资源共享** - 数据库连接、文件句柄、Socket连接等
2. **配置管理** - 应用程序配置只需要一个实例
3. **日志记录** - 日志工具类只需要一个实例
4. **WPF ViewModel** - 避免重复创建造成资源冲突

### 好处

- ✅ 节省内存（只创建一个对象）
- ✅ 避免重复初始化（如 Socket 问题）
- ✅ 全局访问方便
- ✅ 控制对共享资源的并发访问

## 三、实现方式

### 方式1：简单实现（非线程安全）

```csharp
public class Singleton
{
    private static Singleton _instance;

    // 私有构造函数，防止外部创建
    private Singleton() { }

    public static Singleton Instance
    {
        get
        {
            if (_instance == null)
            {
                _instance = new Singleton();
            }
            return _instance;
        }
    }
}
```

### 方式2：线程安全实现（使用 lock）

```csharp
public class Singleton
{
    private static Singleton _instance;
    private static readonly object _lock = new object();

    private Singleton() { }

    public static Singleton Instance
    {
        get
        {
            lock (_lock)  // 加锁，确保线程安全
            {
                if (_instance == null)
                {
                    _instance = new Singleton();
                }
                return _instance;
            }
        }
    }
}
```

### 方式3：最简单实现（C# 推荐）

```csharp
public class Singleton
{
    private static readonly Singleton _instance = new Singleton();

    private Singleton() { }

    public static Singleton Instance => _instance;
}
```

### 方式4：Lazy 实现（延迟加载，推荐）

```csharp
public class Singleton
{
    private static readonly Lazy<Singleton> _lazy =
        new Lazy<Singleton>(() => new Singleton());

    private Singleton() { }

    public static Singleton Instance => _lazy.Value;
}
```

## 四、实现对比

| 方式 | 线程安全 | 延迟加载 | 复杂度 | 适用场景 |
|------|----------|----------|--------|----------|
| 简单实现 | ❌ | ✅ | 低 | 单线程应用 |
| Lock 实现 | ✅ | ✅ | 中 | 多线程环境 |
| 静态初始化 | ✅ | ❌ | 低 | 简单应用，类加载时就初始化 |
| Lazy 实现 | ✅ | ✅ | 低 | 推荐，兼顾性能和安全 |

## 五、关键点总结

| 特点 | 说明 |
|------|------|
| **静态实例变量** | `private static HomeVM _instance;` |
| **私有构造函数** | `private HomeVM() { }` 防止外部 new |
| **静态属性** | `public static HomeVM Instance` 提供全局访问点 |
| **线程安全** | 使用 `lock` 确保多线程环境安全 |
| **延迟加载** | 只在第一次访问时创建实例 |

## 六、实际使用示例

### 基础用法

```csharp
// 创建单例
var homeVM1 = HomeVM.Instance;
var homeVM2 = HomeVM.Instance;

// 验证是同一个对象
Console.WriteLine(homeVM1 == homeVM2);  // True ✓

// 无法从外部创建（编译错误）
// var homeVM3 = new HomeVM();
```

### WPF ViewModel 应用

```csharp
public class HomeVM : ViewModelBase
{
    // 1. 静态实例变量
    private static HomeVM _instance;
    private static readonly object _lock = new object();

    // 2. 静态属性获取实例
    public static HomeVM Instance
    {
        get
        {
            lock (_lock)
            {
                if (_instance == null)
                {
                    _instance = new HomeVM();
                }
                return _instance;
            }
        }
    }

    // 3. 私有构造函数
    private HomeVM()
    {
        SendCommand = new RelayCommand(_ => SendMessage());
        InitSocketManager();
    }

    // 4. 其他代码...
}

// 使用方式（不再是 new，而是获取实例）
HomePage = new RelayCommand(_ => CurrentPage = new HomeView(HomeVM.Instance));
```

## 七、注意事项

### 优点
- ✅ 确保只有一个实例
- ✅ 提供全局访问点
- ✅ 避免资源浪费

### 缺点
- ❌ 测试困难（难以进行单元测试）
- ❌ 违反单一职责原则（既要管理生命周期，又要处理业务逻辑）
- ❌ 可能导致代码耦合

### 最佳实践

1. **只在必要时使用**：不是所有类都需要单例
2. **确保线程安全**：在多线程环境下使用 `lock` 或 `Lazy<T>`
3. **私有化构造函数**：防止外部创建实例
4. **考虑使用依赖注入**：更灵活，易于测试

## 八、相关设计模式

| 模式 | 说明 |
|------|------|
| **单例模式** | 一个类只有一个实例 |
| **工厂模式** | 创建对象的接口，由子类决定实例化哪个类 |
| **依赖注入** | 将对象的创建和使用分离，更容易测试和维护 |
