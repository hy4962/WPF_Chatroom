# Socket 连接原理

## TCP 三次握手（连接建立过程）

服务端是"被动等待"，客户端是"主动连接"，三次握手发生在 `Accept()` 和 `Connect()` 之间，操作系统自动完成。

```
服务端 (ServerManager)                  客户端 (ClientManager)
─────────────────────                   ─────────────────────
① new Socket()                          ① new Socket()
② Bind(IP:Port)  ← 绑定地址
③ Listen(队列)   ← 开始监听
④ Accept()       ← 阻塞等待 ────────── ② Connect(IP:Port) ← 主动连接
         ↓ 连接建立，三次握手完成 ↓
⑤ Receive()      ← 双向收发 ────────── ③ Send() / Receive()
```

**关键：** `Connect()` 成功返回 = 三次握手完成，双端建立连接，之后 `Send`/`Receive` 双向可用。

---

## 服务端流程（已有代码）

```csharp
// ① 创建 Socket
Socket server = new Socket(AddressFamily.InterNetwork, SocketType.Stream, ProtocolType.Tcp);

// ② 绑定地址
server.Bind(new IPEndPoint(ipAddress, port));

// ③ 开始监听
server.Listen(maxNumber);

// ④ 阻塞等待客户端连入（客户端连上来，这里才返回）
Socket client = server.Accept();

// ⑤ 收消息
int bytesRead = client.Receive(buffer);
```

---

## 客户端流程（需要写的代码）

```csharp
// ① 创建 Socket
Socket client = new Socket(AddressFamily.InterNetwork, SocketType.Stream, ProtocolType.Tcp);

// ② 主动连接（服务端的 IP:Port）
client.Connect(serverIP, serverPort);  // 触发三次握手，和服务端 Accept() 对接

// ③ 连接成功后，收发消息
client.Send(bytes);       // 发
client.Receive(buffer);   // 收
```

客户端比服务端少了 `Bind`、`Listen`、`Accept`，多了 `Connect`。

---

## 服务端 vs 客户端对照表

| 步骤 | 服务端 | 客户端 |
|------|--------|--------|
| 创建 Socket | `new Socket()` | `new Socket()` |
| 绑定地址 | `Bind(IP:Port)` | 不需要（系统自动分配） |
| 开始监听 | `Listen(backlog)` | 不需要 |
| 建立连接 | `Accept()` 阻塞等待 | `Connect(IP:Port)` 主动连接 |
| 收消息 | `Receive(buffer)` | `Receive(buffer)` |
| 发消息 | `Send(bytes)` | `Send(bytes)` |

## 本项目中的实现结构

### ServerManager 已有的方法

| 方法 | 职责 |
|------|------|
| `CreationServer(ip, port)` | 创建 Socket + Bind，存入 `_serverDictionary` |
| `ServerStart(ip, port)` | 调用 Listen + 启动 Accept 循环 |
| `AcceptClients(server)` | 循环 Accept，接收客户端存入 `_clientDictionary` |
| `ClientReceiveLoop(client)` | 循环 Receive，通过 `MessageReceived` 事件通知 UI |

### ClientManager 需要写的方法

| 方法 | 职责 | 对应 ServerManager |
|------|------|-------------------|
| `ConnectToServer(ip, port)` | 创建 Socket + Connect，存入 `_clientDictionary` | `CreationServer` + `ServerStart` |
| `ReceiveLoop(Socket)` | 循环 Receive，通过事件通知 UI | `ClientReceiveLoop` |
| `SendMessage(string)` | 发送消息给已连接的服务端 | 服务端无对应，需新增 |

### SettingsVM 需要补的

- 构造函数中初始化 `ClientConnectCommand`
- 写 `ClientConnect_Executed()` 方法，调用 `_clientManager.ConnectToServer(ClientIP, ClientPort)`
- 逻辑和 `CreateServer_Executed()` 对称

## Dictionary 的角色

| Dictionary | 所属 | 存什么 |
|------------|------|--------|
| `ServerManager._serverDictionary` | 服务端 | 自己创建的 Server Socket（监听用） |
| `ServerManager._clientDictionary` | 服务端 | 连接到自己的客户端 Socket |
| `ClientManager._clientDictionary` | 客户端 | 自己创建的 Client Socket（连接远程用） |
| `ClientManager._serverDictionary` | 客户端 | 可选，记录连接的远程服务端信息 |
